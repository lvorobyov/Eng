[glossary]
==Глоссарий

[[st]]Single-threading::
    Однопоточность - последовательное выполнение инструкций.

[[mt]]Multi-threading::
    Многопоточность - одновременное выполнение нескольких последовательностей инструкций.

[[ts]]Thread-safety::
    Потокобезопасность - способность кода программы корректно работать при использовании его одновременно несколькими <<cf,потоками управления>>.

[[re]]Reentrancy::
    Реентерабельность - способность подпрограммы быть вызванной одновременно в разных <<cf,потоках управления>>.

[[cf]]Control-flow::
    Поток управления - последовательность передачи управления цифровым вычислительным устройством от одной инструкции к другой в пределах одного процесса.

[[lc]]Lock::
    Замок - механизм контроля доступа к данным с целью их защиты cite::[978544611314].

[[cfg]]Control-flow graph::
    Граф потока управления - множество всех возможных путей исполнения программы, представленное в виде графа.

[[algo]]Algorithm::
    Алгоритм - порядок действий для решения определённой задачи или правила решения класса задач.

[[cfi]]Control-flow integrity::
    Целостность потока управления - ограничение возможности выполнения программы в пределах заданного графа потока управления.

[[cg]]Call graph::
    Граф вызовов - ориентированный граф, где узлы соответствуют подпрограммам, а рёбра означают вызовы между подпрограммами.

[[ic]]Indirect call::
    Косвенный вызов - вызов процедуры, адрес которой находится в переменной.

[[ib]]Indirect branch::
    Косвенный переход - передача управления инструкции, находящейся по адресу, заданному в переменной.

[[cfa]]Control-flow analysis::
    Анализ потока управления - статический анализ кода с целью построения графа потока выполнения.

[[lb]]Local branch::
    Локальный переход - переход в пределах выполняемой подпрограммы.

[[fe]]Forward-edge::
    Прямой переход - переход к началу подпрограммы.

[[be]]Back-edge::
    Обратный переход - переход к инструкции, следующей за инструкцией вызова подпрограммы.

[[coro]]Coroutine::
    Сопрограмма - средство обеспечения многопоточности путём <<yield,переключения контекстов>> прикладной программой.

[[yield]]Context switch::
    Переключение контекста - сохранение состояния вычислителя в памяти и загрузка из памяти состояния вычислителя, сохранённого при выполнении другой задачи.

[[api]]Application programming interface::
    Интерфейс программирования приложений - набор сигнатур алгоритмов, выполняющих общую задачу, служащих одной цели или обрабатывающих одни и те же данные.

[[term]]Term::
    Термин - это слово или набор слов, которое имеет определённое значение в узкой предметной области.

[[reddit]]Reddit::
    Reddit - сайт, созданный Стивом Хоффманом в 2005 году для обсуждения материалов из сети Интернет.

[[http]]Hypertext transfer protocol::
    Протокол передачи гипертекста - сетевой протокол взаимодействия компьютеров с веб-сервером для предоставления гипертекстовой информации.
